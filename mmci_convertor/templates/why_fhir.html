Why you should use FHIR?
  • The use case of the data is data exchange
  • The user doesn't have a license for SNOMED CT
  • Tech stack of FHIR: server with RESTfull API, and data are stored in Resources - JSON and XML formats
  • The user have additional data that needs conversion as well, then it is usually faster and more developer friendly in FHIR
  • The smaller size of the data set
  • The data should be used for interoperability, not for final storage.
  • The user need to customize the input data and minimize data loss
  • It is necessary to add custom constraints
